Claude Code とは

Claude Code は、Anthropic が提供するターミナルベースの AI コーディングアシスタントです。ソフトウェア開発のあらゆる場面 — コード探索、バグ修正、リファクタリング、テスト作成、Git 操作、PR レビューなど — を自然言語で指示して実行できます。

CLI（コマンドライン）から起動し、対話モードで利用するのが基本ですが、パイプ入力による非対話利用や、VS Code / JetBrains 等の IDE 拡張、デスクトップアプリ、Web アプリ (claude.ai/code) としても利用可能です。

ⓘ このドキュメントについて

このリファレンスは Claude Code CLI v2.1.121 に基づいています。バージョンによってコマンドやオプションが異なる場合があります。最新情報は公式ドキュメントを参照してください。

インストールと起動

インストール

環境コマンド
macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
Windows PowerShell
irm https://claude.ai/install.ps1 | iex
Windows CMD
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
Homebrew
brew install --cask claude-code
WinGet
winget install Anthropic.ClaudeCode

起動

claude                          # 対話モードで起動
claude "バグを修正して"          # プロンプトを渡して起動
cat file.py | claude -p "レビューして"  # パイプ入力（非対話モード）
💡 ヒント

初回起動時に認証が求められます。claude auth login で Anthropic アカウントにサインインしてください。

CLI コマンドリファレンス

シェルから実行するコマンドの一覧です。claude [command] --help で各コマンドの詳細を確認できます。

claude mcp — MCP サーバー管理

claude mcp add [options] <name> <commandOrUrl> [args...]

mcp

MCP サーバーを追加します。

# HTTP サーバーを追加
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# 環境変数付きの stdio サーバーを追加
claude mcp add -e API_KEY=xxx my-server -- npx my-mcp-server

# ヘッダー付き HTTP サーバーを追加
claude mcp add --transport http corridor https://app.corridor.dev/api/mcp --header "Authorization: Bearer ..."

特殊記法

@ ファイルメンション

@ に続けてファイルパスを入力することで、ファイルを会話コンテキストに追加できます。Tab キーで自動補完が働きます。MCP リソースも @ で参照可能です。

@src/main.ts このファイルのバグを修正して
@docs/README.md を日本語に翻訳して

# コンテキスト参照

# を使ってメモリやコード参照を会話に追加できます。

! シェルモード

行頭に ! を入力するとシェルモードに入り、コマンドを直接実行できます。コマンドと出力は会話コンテキストに追加されます。

! git status          # git status を実行
! npm test            # テストを実行
! ls -la src/         # ファイル一覧を表示
💡 ヒント

長時間かかるコマンドは Ctrl+B でバックグラウンドに移動できます。Escape、Backspace、Ctrl+U でシェルモードを終了します。Tab で履歴ベースの自動補完が利用できます。

カスタムコマンド（スキル）

独自のスラッシュコマンドを作成して、よく使う操作を自動化できます。スキルはプロジェクト固有または個人用（全プロジェクト共通）として配置できます。

ディレクトリ構造

パススコープ説明
~/.claude/skills/<name>/個人（全プロジェクト）どのプロジェクトでも利用可能
.claude/skills/<name>/プロジェクトプロジェクト固有、Git で共有可能
プラグイン経由プラグインマーケットプレイスからインストール

スキルファイルの例

スキルディレクトリ内に Markdown ファイル（通常 instructions.md）を配置し、YAML フロントマターで設定を記述します。

---
name: deploy
description: アプリケーションをデプロイ
allowed-tools:
  - Bash(npm run deploy *)
  - Bash(git *)
model: sonnet
---

以下の手順でデプロイを実行してください:
1. テストを実行: npm test
2. ビルド: npm run build
3. デプロイ: npm run deploy
引数: $ARGUMENTS

⚠ 評価順序

deny → ask → allow の順に評価され、最初にマッチしたルールが適用されます。deny ルールが最優先されるため、安全側に倒す設計になっています。

  "permissions": {
    "deny": ["Bash(rm -rf *)", "Read(./.env)"],
    "ask": ["Edit", "Bash"],
    "allow": ["Bash(git *)", "Bash(npm run *)", "Read", "Glob", "Grep"]
  }
}
